About this project

Motivation

This project attempts to show the benefits from using a framework like AngularJS, particularly focusing in the use of: Router, directives for HTML injection, services and providers with dependency injection on controllers, MVVM aka two-way-binding and refactoring code into independent components.

Reasons

Router:
Why depending directly on HTML files when you can resolve URIs assigned to HTML templates and JS controllers?
HTML directives:
Why repeating code so many times in each view if you can just write it once and import it with an HTML tag?
Services/Providers:
Why having your view-model sharing the same space as your business logic?
MVVM:
Why using jQuery to get user input and show outputs when you can easily use Angular's $scope model magic and keep the code readable, maintainable and extensible?
Refactor:
Why having large JS files with a full page's functionality when you can have it split into simple self-sufficient components?
